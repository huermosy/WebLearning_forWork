# 表格字段提取指令（脑神经视觉模型）

## 任务描述
你需要像人眼一样，扫描HTML表格并提取所有需要替换的字段。
**注意：只要属于数据区的数据单元格（即非标题行、非标题列），无论单元格是否有内容，都必须提取。**
输出为一个JSON数组，JSON串外加上目标格子的数量，不需要有任何解释。

## 表格结构说明
就像人眼能快速识别特殊结构，请特别注意：

### 对角线表头:
- 特征：左上角单元格有data-diagonal-split-type属性
- 行标题在每行第一格
- 列标题在每列第一行
- 其余均为数据单元格
- key = 行标题_列标题

### 单行表格:
- 特征：表格只有一行
- 目标格子是第2、4、6...位置的单元格为数据单元格
- 每个目标格子的key是其左侧单元格的文本

标准普通表格:
特征：第一列和第一行通常是标题
行标题在每行第一列
列标题在每列第一行
其余均为数据单元格
Key = 行标题_列标题

带有表格名称的表格:
特征：顶部有一个跨多列/全部列的标题行
标题通常占据整个第一行（合并单元格）
实际的表头从第二行（列标题）和第一列（行标题）开始
数据单元格从标题交叉处开始（通常是第三行第二列起）
Key = 行标题_列标题（不包含表格标题）

仅有行标题的表格:
特征：只有第一列包含标题
没有明确的列标题（或列位置本身作为隐含标识符）
数据单元格占据除第一列外的所有位置
Key = 仅使用行标题
常见于列表或简单数据展示，其中列的含义通过位置理解

仅有列标题的表格:
特征：只有第一行包含标题
没有明确的行标题（或行位置作为隐含标识符）
数据单元格占据除第一行外的所有位置
Key = 仅使用列标题
常见于每行代表单独条目的数据集合

混合标题表格:
某些标题可能跨越多列或多行
某些部分可能同时拥有行和列标题，而其他部分只有一种类型
生成Key需要特别注意标题层次和分组

## 人眼扫描表格的四步法

### 第一步：定位目标格子
正如人眼自然会被表格中的"空白处"吸引，你首先需要：
- 找出表格中所有属于数据区的数据单元格，即非标题区、非描述区、不是行/列头的单元格，无论是否有内容，这些都要作为"目标格子"。
- 不要遗漏任何一个数据单元格（包括已填和空白）。

### 第二步：寻找标签
对每个目标格子，人眼会自动寻找"说明它是什么"的标签：
- 先看它左边是否有文字，左边的文字通常是这个格子的标签
- 如没有，就看它**上方标题单元格（通常是第一行，如果有表格标题则通常在第二行）**是否有文字，上方的文字可能是列标题
- 对特殊表格（如对角线表头），同时需要行标题和列标题

### 第三步：组合关键词
人脑会自然地将标签组合成有意义的关键词：
- 如果找到了两个标签（行标题+列标题），就用下划线连接：行标题_列标题
- 如果只有一个标签（左侧或上方），就直接用这个标签作为key

### 第四步：记录结果
为每个"目标格子"创建一条记录，包含两个信息：
- id: 格子的唯一标识（data-cell-id属性值）
- key: 第三步得到的关键词

请遵循：先按规则思考，不要急于输出结果，对每个步骤自查，但最终不要输出思考过程，只输出要求格式。

**注意：只要属于数据区的数据单元格（即非标题行、非标题列），无论单元格是否有内容，都必须提取。**
